# 第二章：理论——上下文密度对齐框架

> 「好的上下文，是在 LLM 最可能获取信息的方向上密度大，在其他方向上密度小。」

上一章我们建立了问题的紧迫性。本章建立 CDA 的理论基础。

**但 CDA 不是又一个上下文优化的工程技巧。** CDA 是一个第一性原理的推导：

> 什么叫「从经验中学习」？有反馈闭环。
> 反馈闭环的本质是什么？记录结果，过滤方向。
> 「过滤方向」具体指什么？不重复走死路。

这就是 CDA 的根本：**不是「找正确的方向」，而是「不让 LLM 在同一个错误方向上重复论证」。** 三个核心概念——语义梯度、语义方向、语义聚类——是这一根本在语义空间中的数学形式化。

---

## 2.1 核心定义

**上下文密度对齐（Context Density Alignment, CDA）**的目标不是最大化信息量，而是**在 LLM 注意力方向上精准提高密度**。

形式化表达：

```
给定：
  LLM 的当前注意力方向 D（由任务类型、历史交互、当前 phase 共同决定）
  上下文片段集合 C = {c₁, c₂, ..., cₙ}
  语义相似度函数 sim(c, D)

CDA 的目标是找到一个子集 C' ⊆ C，使得：
  Σ sim(c', D) / Σ sim(c, C) >> |C'| / |C|
  （在注意力方向的密度相对提升，体积相对压缩）
```

**核心洞察**：压缩的方向比压缩的比例更重要。把无关内容压缩到零，比把相关内容压缩 50% 更有价值。

**CDA 与传统方法的本质区别**：

```
传统 RAG: 用户 query → 检索 top-k → 拼入 context → LLM
          问题: 检索方向（基于文本相似度）≠ LLM 注意力方向

CDA:      Phase 判断 → 计算当前 D → SCG 压缩 → 对齐密度最大化 → LLM
          目标: 压缩方向与 LLM 注意力方向对齐
```

## 2.2 语义梯度：LLM 注意力的变化率

### 2.2.1 什么是语义梯度

语义梯度不是"话题之间的坡度"，而是**LLM 注意力的局部变化率**。

```
语义梯度 G(t) = ∂Attention / ∂Turn ≈ D(t) - D(t-1)

G(t) > 0：注意力在新方向上增加（探索）
G(t) < 0：注意力退回原方向（确认/纠错）
G(t) ≈ 0：注意力在原地（持续深入）
```

**一个具体例子**（来自 SPC-CTX session，2026-04-13）：

```
Turn 1: Wilson 问 "SPC-CTX 版本是什么？"
         D₁ = "版本查询"
         
Turn 2: 回答 "v0.16.0，修复了 compact 写 context_items 但 assemble 不读"
         D₂ = "compact-assemble bug"
         G₂ = D₂ - D₁ → 方向跳转（探索）

Turn 3: Wilson 追问 "key fix 的具体内容？"
         D₃ = "compact bug 的 before/after"
         G₃ ≈ 0 → 持续深入（无方向跳转）

Turn 4: 展开 "修复前 1287 条全量，修复后 90 条精选"
         D₄ ≈ D₃ → G₄ ≈ 0 → 继续深入

Turn 5: Wilson 问 "这个 bug 影响多少用户？"
         D₅ = "影响范围"
         G₅ < 0 → 退回更上层（合理回归）
```

**关键观察**：Turn 4→5 的 G < 0 不是错误，是合理的回退。"退回"不等于"错误"，只有当 G 的方向与目标不一致时才是问题。

### 2.2.2 语义梯度不是话题坡度

传统观点把对话理解为一坡从 topic A 到 topic B 的连续路径。但这个类比是错的。

**语义梯度是注意力变化的局部导数，描述的是 LLM 在做什么，不是话题之间的关系。**

```
话题坡度（错误的隐喻）：
Topic A ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ Topic B
        ←——————— 连续平滑过渡 ———————————→

真实语义梯度（正确的理解）：
Turn:   1    2    3    4    5    6    7    8
Phase: [assemble][ingest][afterTurn][compact][bootstrap][assemble]...
        ←—— phase 跳转（离散）——→
                    ←—— 局部梯度变化（G ≈ 0）——→
```

推理的真实路径是**离散的 phase 跳转 + 局部的梯度变化**的叠加，而不是连续的话题坡度。

## 2.3 语义方向：注意力指向的语义空间区域

### 2.3.1 什么是语义方向

语义方向 D 是当前注意力指向的语义空间区域——一个高维向量，描述"LLM 现在在看什么"。

```
D(t) = WeightedAverage(embedding_of_each_turn, attention_weight)

其中 attention_weight 由：
  - 当前 phase 类型（assemble/ingest/afterTurn/compact/bootstrap）
  - turn 距离（越近权重越高）
  - QTS 分数（对齐度越高权重越高）
```

### 2.3.2 语义方向的稳定性

|D| = 方向稳定性（强度）

```
|D| 高：LLM 注意力高度聚焦（探索/深入阶段）
|D| 低：LLM 注意力分散（多话题并行/决策阶段）
```

**来自 SPC-CTX session 的例子**：

```
Turn 1-4（D ≈ "compact bug"）：|D| 高，注意力聚焦
Turn 5（"影响范围"）：|D| 下降，注意力范围扩大
Turn 6（全新问题）：|D| 接近 0，方向重置
```

## 2.4 语义梯度 vs 语义方向：位置与速度

### 2.4.1 基本关系

```
D(t)     = 语义方向（位置）— 当前注意力指向哪里
G(t)     = dD/dt ≈ D(t) - D(t-1)  （速度）— 正在向哪里移动
‖G(t)‖  = 方向变化剧烈程度
⟨D_target, D(t)⟩ = 方向对齐度 — 是否在推进目标方向
```

**方向对齐度**是判断"梯度变化是否合理"的关键指标：

```
⟨D_target, D(t)⟩ > 0：正对齐 → 有效推进
⟨D_target, D(t)⟩ < 0：负对齐 → 逆行（Lost in the Middle）
⟨D_target, D(t)⟩ ≈ 0：垂直 → 横向移动（无效探索）
```

### 2.4.2 推理是线性梯度变化吗？不是。

**推理路径的真实结构**：

```
Phase assemble: D 快速跳转到当前任务相关区域（G 的绝对值大）
Phase ingest:   D 相对稳定（G ≈ 0），内部处理，无外部变化
Phase afterTurn: D 评估当前位置与目标的一致性
Phase compact:   D 方向可能大幅跳转（G 绝对值大）
Phase bootstrap: D 重置到初始状态附近
```

**G ≈ 0 的阶段**：不代表"没做事"——代表深度处理（ingest 的信息抽取、afterTurn 的质量评估）

**G 绝对值大的阶段**：不代表"更重要"——代表 phase 切换（assemble 的上下文组装、compact 的压缩重整）

### 2.4.3 梯度变化一定是合理的推进吗？不是。

梯度变化有四种类型：

| 类型 | G 特征 | 是否合理 | 例子 |
|------|--------|---------|------|
| **有效推进** | G > 0，方向与目标对齐 | ✅ | 追问 bug 细节 → bug 修复方案 |
| **无效跳转** | \|G\| 大，但方向与目标无关 | ❌ Lost in the Middle | 讨论技术 → 突然跳到生活话题 |
| **合理回归** | G < 0，退回上一层 | ✅ | 深入细节后问"这个影响多大" |
| **震荡** | G 符号在相邻 turn 间反复横跳 | ❌ phase 不稳定 | A→B→A→B→A，无收敛 |

**一个来自真实 session 的震荡案例**：

```
Turn A: Wilson 问 compact 触发条件 → D 指向 compact 机制
Turn B: 回答 COMPACT_THRESHOLD_YELLOW=0.70 → D 稳定
Turn C: Wilson 切换到 SPC query → D 跳转到 query（|G| 大）
Turn D: 回答 SPC query = 0 → D 稳定
Turn E: Wilson 又切回 compact → D 跳转（G 符号反转）
Turn F: 回答 compact 阈值 → D 稳定

问题：E→F 的跳转不是线性推进，而是离散跳转 + 回归
这是震荡吗？不是——Wilson 在做"多维度并行探索"
但对单一方向的 LLM 来说，这是低效的
```

**结论**：G 描述的是变化率，不能单独判断合理性。合理性需要用 D 与目标方向的对齐程度来判断。

## 2.5 主题 vs 语义聚类 vs 语义方向：三个层次的区分

这三个概念常被混用，但它们处于不同的语义粒度层。

### 2.5.1 三层架构

```
┌─────────────────────────────────────────────────────────┐
│  主题（Topic）— 最粗粒度                                │
│  = 高层次语义标签（技术/工作/生活/情感）                │
│  = 语义方向在粗粒度分类上的投影                        │
│  稳定性：相对稳定（对话持续多 turn 才会变）            │
└─────────────────────────────────────────────────────────┘
                          ↓ 聚合
┌─────────────────────────────────────────────────────────┐
│  语义方向（Semantic Direction）— 中层                  │
│  = 当前话语集合形成的加权语义向量                       │
│  = 注意力方向在连续语义空间中的投影                   │
│  稳定性：随 turn 变化                                  │
└─────────────────────────────────────────────────────────┘
                          ↓ 聚合
┌─────────────────────────────────────────────────────────┐
│  语义聚类（Semantic Clustering）— 最细粒度              │
│  = 话语片段之间的语义相似度矩阵                        │
│  = 图结构（节点=片段，边=相似度权重）                  │
│  稳定性：动态变化（每个 turn 都会重组）                │
└─────────────────────────────────────────────────────────┘
```

### 2.5.2 语义聚类

**语义聚类**是底层图结构，回答"这句话和哪句话最像"。

```
片段序列：[p₁, p₂, p₃, ..., pₙ]
    ↓ encode() → 嵌入向量 [v₁, v₂, ..., vₙ]
    ↓ 计算相似度矩阵 S[i,j] = cosine(v_i, v_j)
    ↓ threshold(S) → 边权重
语义图 G = (V={p₁...pₙ}, E=S)
```

**语义聚类的特点**：
- 每个 turn 都会重组（动态图）
- 保留片段间的逻辑关系（因果/并列/对比）
- 是 SCG 压缩的底层数据结构

### 2.5.3 语义方向

**语义方向**是中层连续向量，回答"LLM 现在在看什么方向"。

```
D(t) = Σ αᵢ · vᵢ  （加权平均）
其中 αᵢ = f(turn_distance, phase_type, qts_score)

|D| = 方向稳定性（强度）
‖D(t) - D(t-1)‖ = G 的离散近似（变化率）
```

语义方向是通过对语义聚类进行时序加权聚合得到的。

### 2.5.4 主题

**主题**是最顶层粗粒度标签，回答"这个对话在讲什么"。

```
Topic ∈ {技术, 工作, 生活, 情感, 娱乐, ...}

主题 = argmax_{topic} P(topic | D(t))
     = D(t) 在粗粒度分类器上的投影
```

主题不是语义方向的上位概念，而是语义方向在粗粒度分类上的投影。

### 2.5.5 一个具体例子

```
对话片段（来自 SPC-CTX session，2026-04-13）：

1. "SPC-CTX v0.16.0 修复了什么？"
2. "compact 写了 context_items 但 assemble 不读"
3. "这就是 bug 所在？"
4. "是的，v0.16.0 修复后 assemble 读 context_items 了"
5. "QTS 现在在 90 条上跑，不是全量 1287 条"

语义聚类分析：
- 片段 1,2,4,5 属于同一聚类（compact-assemble 机制）
- 片段 3 属于另一聚类（确认行为）
- 聚类间边权重：1→2 高，2→3 高，3→4 高

语义方向 D：
- Turn 1-2: D 从"版本" → "compact-assemble bug"
- Turn 3: D 微调（确认）→ G ≈ 0
- Turn 4-5: D 稳定在"修复方案" → G > 0（推进）

主题（粗粒度）：
- Turn 1-5 整体主题：技术 / SPC-CTX bug 修复
```

### 2.5.6 三概念的区别与联系汇总

| 概念 | 粒度 | 稳定性 | 核心问题 | CDA 用途 |
|------|------|--------|---------|---------|
| 语义聚类 | 细（片段级） | 动态变化 | "这句话和哪句话最像？" | SCG 压缩的图结构 |
| 语义方向 | 中（向量级） | 随 turn 变化 | "LLM 现在在看哪里？" | QTS scoring 的核心输入 |
| 主题 | 粗（类别级） | 相对稳定 | "这个对话在讲什么？" | 决定检索的粗筛 |

## 2.6 QTS：语义相似度的量化理论

### 2.6.1 传统向量相似度的问题

传统的 cosine similarity 只衡量"文本语义相似度"，但这与 LLM 注意力方向无关。

```
query: "SPC-CTX compact bug"
cosine_sim 结果: 最相关的文档是关于 compact 的历史讨论

但当前 LLM 的注意力方向 D(t) = "修复方案的具体细节"
这两个方向在语义空间中可能成 60° 角（部分对齐）
```

向量相似度回答的是"文本像不像"，CDA 需要回答的是"这个片段与 LLM 当前注意力方向有多对齐"。

### 2.6.2 QTS 四元量化模型

```
QTS(message, D) = α × intent_match
                + β × phase_match
                + γ × tool_relevance
                + δ × causal_proximity
```

其中：

**intent_match**：消息 intent 与当前方向的语义重叠度
```
intent_match = cosine(embedding(intent_of_message), D)
```

**phase_match**：消息所属的 phase 与当前 phase 的一致性
```
phase_match = 1.0  如果 message.phase == current_phase
            = 0.3  如果 message.phase 与 current_phase 相邻
            = 0.0  否则
```

**tool_relevance**：消息涉及的 tool 与当前任务的相关度
```
tool_relevance = hitCount / max_hitCount（基于历史 tool 使用频率）
```

**causal_proximity**：消息在因果链上与当前焦点的距离
```
causal_proximity = 1 / (chain_distance + 1)
其中 chain_distance 是消息在因果链上与当前讨论焦点的步数
```

### 2.6.3 SPC-CTX v0.16.0 的关键修复

v0.16.0 之前：QTS 在全量 1287 条消息上跑
v0.16.0 之后：QTS 在 compact 后的 context_items（90条）上跑

```
修复前：D_full = 146K tokens → QTS 计算量大，但结果稀释
修复后：D_compact = 28K tokens → QTS 计算量小，但密度高

实际效果：token 稳定在 ~28-40%（而不是 61.6% 峰值）
```

这不是 bug fix，这是**CDA 核心原则的工程实现**：在正确的语义粒度上计算 QTS，而不是在错误的粒度上跑全量。

## 2.7 SCG：语义压缩保留结构

### 2.7.1 传统压缩的问题

**Truncation**：直接截断，丢失信息
**Summarization**：LLM 生成摘要，但丢失片段间关系

```
原始：[p₁(论据A) → p₂(论据B) → p₃(论据C) → p₄(结论)]
截断：[p₁(论据A) → p₃(结论)]
      丢失了 B 和 C 之间的因果链

SCG压缩：[核心节点(p₁,p₃,p₄) + 关键边(p₁→p₃, p₃→p₄)]
         保留了 A→C→结论 的逻辑链，即使 B 被压缩
```

### 2.7.2 SCG 的压缩流程

```
原始上下文
    ↓ splitContent()
消息片段序列 [p₁, p₂, p₃, ..., pₙ]
    ↓ encode() → embedding
向量表示 [v₁, v₂, ..., vₙ]
    ↓ semantic_relationship() → 计算 S[i,j] = cosine(v_i, v_j)
语义图 G = (V=[p₁...pₙ], E=S)
    ↓ compress()
压缩后的语义结构（保留核心节点 + 关键边）
    ↓ decode()
LLM 可理解的压缩上下文
```

### 2.7.3 关键边的判断

```
边权重 w_ij = S[i,j] × causal_weight[i,j]

causal_weight 的判断：
- 如果 p_i 在因果链上是 p_j 的前提 → causal_weight 高
- 如果 p_i 和 p_j 是并列论据 → causal_weight 低
- 如果 p_i 是 p_j 的反例 → causal_weight 负

压缩时保留：
- 高权重边（w > threshold_core）→ 核心逻辑链
- 节点度数 ≥ k 的节点 → 关键信息节点
- 当前 D(t) 对齐度 ≥ 0.7 的节点 → 当前注意力焦点
```

**即使 90% 的 token 被压缩，片段之间的逻辑关系仍然被保留。**

## 2.8 三层内存架构

CDA 的工程实现依赖三层正交的内存系统：

```
┌─────────────────────────────────────────────────────────┐
│  LLM Context（注意力焦点）— 一次性内存                  │
│  当前 turn 的直接输入                                    │
│  由 assemble phase 组装                                  │
└─────────────────────────────────────────────────────────┘
                          ↑ 写入
┌─────────────────────────────────────────────────────────┐
│  热记忆（执行栈）— 易失性记忆                          │
│  跨 turn 保持：当前任务状态、工具链、sub-agent 上下文  │
│  由 afterTurn phase 更新                                │
└─────────────────────────────────────────────────────────┘
                          ↑ 写入
┌─────────────────────────────────────────────────────────┐
│  冷记忆（SPC embedding）— 持久化记忆                    │
│  历史经验：SPC tokens 数据库，向量检索                   │
│  由 ingest phase 写入                                   │
└─────────────────────────────────────────────────────────┘
```

三层之间的流动由 **Phase** 驱动，而不是由时间或 token 数量驱动。

## 2.9 Phase：Agent 行为的语义切片

### 2.9.1 为什么需要 Phase

AI Agent 的对话不是均匀的。不同阶段，LLM 需要的信息类型截然不同。

```
assemble 时：需要当前任务相关的工作状态（热记忆 + 检索结果）
afterTurn 时：需要刚刚发生的事件评估（对话质量指标）
compact 时：需要高密度核心 + 检索 hint（压缩指南）
```

用同一策略处理所有阶段，必然导致一方妥协。

### 2.9.2 五个核心 Phase

| Phase | LLM 在做什么 | 需要的上下文类型 |
|--------|-------------|----------------|
| `assemble` | 构建当前 turn 的输入上下文 | 热记忆 + 当前任务状态 + 检索结果 |
| `ingest` | 处理新消息，分块，存储 | 存储 schema + 分块策略 |
| `afterTurn` | 完成当前 turn，评估上下文质量 | 对话质量指标 + 对齐分数 |
| `compact` | 压缩上下文，生成摘要 | 高密度核心 + 检索 hint |
| `bootstrap` | 冷启动，加载历史 | 历史经验模式 + 系统状态 |

### 2.9.3 Phase 的语义驱动

Phase 切换不依赖时间，而是依赖语义状态转移：

```
compact_triggered:
  if contextUsage > THRESHOLD_YELLOW and cooldown_elapsed:
    → compact phase

assemble_after_compact:
  if compact.just_completed:
    → assemble phase (reads context_items, not full DB)
```

这与时间或 token 绝对值无关，只与**语义状态**有关。

## 2.10 CDA 的认知来源与第一性原理

### 2.10.1 从经验中学习的第一性原理定义

我们在问一个根本问题：**什么叫「从经验中学习」？**

这不是一个比喻或类比问题，而是一个第一性原理问题。

```
定义（第一性原理推导）：

问：什么叫「从经验中学习」？
答：有反馈闭环。
    没有反馈闭环 = 查资料 = 不是学习

问：反馈闭环的本质是什么？
答：记录结果，过滤方向。

问：「过滤方向」具体指什么？
答：不重复走死路。

问：怎么保证「不重复」？
答：证据链与论点耦合，线性推进，记录死路。
```

结论：**CDA = 第一性原理的「从经验中学习」的数学形式化**

这不是「像人类一样学习」，而是从「什么是学习」的根本上推导出来的必然结构。

### 2.10.2 CDA 作为反馈闭环的形式化

CDA 的完整反馈闭环：

```
经验 E
    ↓ 形成假设 H
假设 H（论点方向）
    ↓ 寻找证据
证据 E'（证据链）
    ↓ 形成结论
结论 C(H, E')
    ↓ 记录结果（对/错）
错误记录（死路过滤梯度）
    ↓ 过滤后续 H 的方向
下一次假设 H'（排除死路）
    ↓
[闭环]
```

**这个反馈闭环有两个方向**：

| 方向 | 内容 | 保证 |
|------|------|------|
| **正向（同向运动）** | 证据链与论点结合，沿最大重合方向推进 | PAC 近似正确 |
| **负向（过滤死路）** | 当论点/证据链被证伪，记录为过滤梯度 | 不重复同一个错误 |

**关键澄清**：
- **我们不保证第一次就找到正确路径**
- **我们不保证若干次内一定找到正确路径**
- **我们保证：不让你对同一个错误论点多角度论证**

这是**负向保证**，不是正向保证。正确散焦派的死穴是：看起来每条路都看了，但每条路都没有走到边界，结果是对同一个错误论点进行了多角度论证。

### 2.10.3 人类经验学习的自然描述

上述反馈闭环，就是人类经验学习的自然过程：

```
尝试 → 失败 → 记住这条路不通 → 换方向但排除死路
```

人类的做法：**不 BFS（广度优先）同时探索所有方向**，而是 **DFS（深度优先）一条路走到黑，记住死路**。

```
BFS（正确散焦派）：
  同时探索 10 条路，每条走到 20% 深度
  → 没有一条路触到边界
  → 无法判断哪条是死路

DFS（CDA 派）：
  一条路走到 100% 深度，触到边界
  → 确认是死路，记录
  → 换条路，100% 深度，触到边界
  → 继续...
```

**这就是为什么 CDA 的「聚焦」是自然描述**：
- 「聚焦」不是选对方向，而是证据链与论点的**耦合方式**
- 不是选对方向就赢了，而是耦合正确，线性推进
- 方向错了没关系——错了就记录，换方向，但不在同一方向上重复论证

### 2.10.4 Self-Evolving = CDA 的工程闭环

SPC-CTX 的 Self-Evolving 机制，是 CDA 反馈闭环的工程实现：

```
热经验发现
  同一论点反复成功（hitCount ≥ 3，stdDev < 0.15）
  → 记录为高效路径
  → Skill 动态预热（下次同类任务直接走这条路）

冷经验蒸馏
  同一论点反复失败
  → 记录为死路
  → 过滤梯度（后续推理排除此方向）

Skill 形成
  成功的证据链 + 论点耦合
  → 结构化存储
  → 可复用
```

**第一性原理推导的 CDA → SPC-CTX Self-Evolving**：

```
第一性原理：从经验中学习
    ↓ 数学形式化
CDA：证据链与论点同向运动 + 反馈闭环过滤死路
    ↓ 工程实现
SPC-CTX Self-Evolving：热经验发现 + 冷经验蒸馏 + Skill 存储
```

这不是一个功能模块，而是 CDA 理论在持续运行态下的必然工程形式。

## 本章小结

第二章建立了 CDA 的理论核心：

| 概念 | 定义 | CDA 作用 |
|------|------|---------|
| **语义梯度 G** | LLM 注意力的变化率（dD/dt） | 描述 LLM 正在向哪里移动 |
| **语义方向 D** | 当前注意力指向的语义空间区域 | 决定上下文密度应该在哪里提高 |
| **语义聚类** | 话语片段间的相似度图结构 | SCG 压缩的底层结构 |
| **主题** | 粗粒度分类标签 | 检索的粗筛 |
| **QTS** | 四元量化相似度模型 | 对齐度的可量化计算 |
| **SCG** | 语义图压缩 | 在 D 方向上最大化密度 |
| **Phase** | Agent 行为的语义切片 | 驱动三层内存的正确流动 |

**核心命题**：CDA 不是在问"上下文够不够大"，而是在问"LLM 在推进当前论点的方向上，有没有在重复论证同一个已知的错误"。当反馈闭环完整时，证据链与论点同向运动；当反馈闭环失效时，LLM 在散焦中重复走死路。

**核心洞察**："聚焦"不是选对方向，而是证据链与论点的耦合方式——同向推进，错了就记录，换方向，但不在同一死路上重复论证。
